  <h2>常见问题集</h2>

  <details open>
    <summary><h3>什么是 Aiken？</h3></summary>
    <p>
      Aiken 是一门现代化的 <strong>编程语言与工具组，用于开发
      智能合约</strong> 于 <a href="https://cardano.org/"
      target="_blank">Cardano</a> 区块链上。它致力于提升稳健性
      与开发者体验。
      <br/>
      Aiken 从许多现代语言中汲取灵感，例如 Gleam、Rust
      与 Elm，这些语言以友好的错误消息与整体
      卓越的开发者体验而闻名。我们相信 Cardano 值得拥有一门具备这些功能的专属
      语言，并与社区共同公开开发。
    </p>
  </details>

  <details>
    <summary><h3>为什么要建立另一个智能合约平台？</h3></summary>
    <p>
      我们热爱函数式编程。然而，我们不禁认为现有的
      Haskell 框架在开发者体验方面<i>不太理想</i>，并觉得我们可以对此做出改进。
      因此，我们在构建 Aiken 时，制定了一套始终铭记在心的共同目标：
    </p>
    <ul>
      <li className="icon-coffee">编写智能合约应该要 <strong>简单且
      安全</strong>。您应该能够在几分钟而非几天内上手，并快速建立对链上代码符合预期运作的信心。</li>

      <li className="icon-heart">我们追求完整且令人愉悦的体验。现代化的
      区块链值得拥有一套现代化的智能合约工具组。<strong>您应该
      在编写合约时感到
      高效且顺手。</strong> 这包括
      编辑器集成（如 LSP 和 tree-sitter）、美观的错误
      消息、快速且直观的测试反馈循环，以及易于产出的
      文档。</li>

      <li className="icon-fast-forward">我们希望<strong>尽可能减少配置</strong>。它应该开箱即用，并与社区建立合理的、具备明确观点的惯例。</li>

      <li className="icon-grid">我们希望采用<strong>模块化设计</strong>，这样就可以根据需要挑选组件。就像 Unix
      哲学一样。Aiken 只是宏大蓝图中的一部分；虽然我们
      坚信它是一个很棒的工具，但我们希望鼓励其他方法
      并尽可能地支持互操作性。 </li>
    </ul>
  </details>

  <details>
    <summary><h3>我以为 Cardano 智能合约必须用 Haskell 编写？</h3></summary>
    <p>
      这是一个常见的误解。目前的 Cardano 节点实现确实是用 Haskell 编写的。节点内置用于执行智能合约的虚拟机也是用 Haskell 实现的。但这并不代表智能合约虚拟机执行的是 Haskell。
      <br/>
      该虚拟机是一个语言解释器，用于执行一种称为 <i>「无类型 Plutus 核心」(Untyped Plutus Core)</i>（简称 <code>UPLC</code>），通常简称为 <i>「Plutus」</i>。
      <br/>
      然而，UPLC 并非开发者需要手写的语言。相反地，它是一个编译目标（就像全球信息网的 WebAssembly）。奇怪的是，直到最近，唯一能从高阶语法产生 UPLC 的既定框架称为 <i>「Plutus-Tx」</i>，且恰好是一个 Haskell 框架。
      <br/>
      Aiken 通过提供一个能直接编译为 UPLC 的现代化框架，改变了游戏规则。
    </p>
  </details>

  <details>
    <summary><h3>我可以使用 Aiken 编写链下/后端代码吗？</h3></summary>
    <p>
      我们的主要目标是改善 Cardano 区块链的
      <strong>智能合约开发体验</strong>。通过将 Aiken 开发为一种
      专用语言，我们可以使其功能更符合链上开发的
      实际需求。
      <br/>
      我们希望保持该语言的简洁与易于管理。在去中心化
      架构中，智能合约带来了巨大的挑战，单一缺陷就可能演变成
      数百万美元的金融漏洞。此外，
      链上代码通常仅占整个 DApp 源代码的一小部分。
      虽然编写合约应尽可能简单直接，但
      智能合约应针对审查、审计和静态
      分析进行优化。
      <br/>
      因此，Aiken <strong>并非</strong> 旨在成为一种通用
      语言。相反，它专注于 Cardano，并致力于提供一套高质量的
      工具组，以便自信地开发可靠的智能合约。
    </p>
  </details>
