# 命令行工具 (CLI)

## 评估 (Evaluation)

让我们考虑以下基础程序：

<p align="right">
  <strong>program_1.uplc</strong>
</p>

```uplc
(program
  1.0.0
  [ [ (builtin addInteger) (con integer 16) ] (con integer 26) ]
)
```

我们可以使用 Aiken 的 CLI 评估此程序：

```sh
aiken uplc eval program_1.uplc
```

```json
{
  "result": "(con integer 42)",
  "cpu": 321577,
  "mem": 602
}
```

输出显示了评估结果 (`42`) 以及该程序的执行成本（包括 CPU 和内存使用率）。

请注意，该命令也接受参数。例如，如果我们将程序修改为接收参数的函数：

<p align="right">
  <strong>program_2.uplc</strong>
</p>

```uplc
(program
  1.0.0
  (lam x [ [ (builtin addInteger) (con integer 16) ] x ])
)
```

您可以通过在 `eval` 命令后附加参数，直接通过 Aiken 提供参数：

```sh
aiken uplc eval program_2.uplc "(con integer 26)"
```

```json
{
  "result": "(con integer 42)",
  "cpu": 390577,
  "mem": 902
}
```

## 格式化 (Formatting)

由于手写 UPLC 可能是一项繁琐的任务，Aiken 提供了一种通过 `fmt` 命令自动格式化 UPLC 程序的快速方法。默认情况下，该命令会覆盖输入文件，但您也可以使用 `--print` 仅将结果打印到 stdout。例如：

```sh
aiken uplc fmt program_2.uplc --print
```

```uplc
(program
  1.0.0
  (lam x [ [ (builtin addInteger) (con integer 16) ] x ])
)
```

## 与二进制编码相互转换

到目前为止，我们一直使用高阶语法来表示 UPLC 程序。然而，在实践中，当 UPLC 程序提交到链上时，它们会被编码为紧凑的二进制字符串（使用 [flat](http://quid2.org/docs/Flat.pdf)）。

Aiken 提供了通过 `encode` 和 `decode` 命令将高阶 UPLC 程序转换为低阶 flat 编码（或反之亦然）的工具。例如：

```sh
aiken uplc encode program_1.uplc --hex
```

```
01000023370090100009
```

`encode` 命令将所有内容打印到 stdout，`--hex` 将字节转换为十六进制编码字符串，以便我们阅读。

<Callout type="info">
 `aiken uplc encode program_1.uplc > program_1.flat`

將原始 "flat" 字节发送到名为 `program_1.flat` 的文件中。
</Callout>

从那里，可以使用 `decode` 从 flat 程序恢复 UPLC 高阶语法：

```sh
aiken uplc decode program_1.flat
```

```uplc
(program
  1.0.0
  [ [ (builtin addInteger) (con integer 16) ] (con integer 26) ]
)
```
