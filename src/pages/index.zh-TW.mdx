  <details>
    <summary><h3>我以為 Cardano 智能合約必須用 Haskell 編寫？</h3></summary>
    <p>
      這是一個常見的誤解。目前的 Cardano 節點實作確實是用 Haskell 編寫的。節點內建用於執行智慧合約的虛擬機也是用 Haskell 實作的。但這並不代表智慧合約虛擬機執行的是 Haskell。
      <br/>
      該虛擬機是一個語言直譯器，用於執行一種稱為 <i>「無類型 Plutus 核心」(Untyped Plutus Core)</i>（簡稱 <code>UPLC</code>），通常簡稱為 <i>「Plutus」</i>。
      <br/>
      然而，UPLC 並非開發者需要手寫的語言。相反地，它是一個編譯目標（就像全球資訊網的 WebAssembly）。奇怪的是，直到最近，唯一能從高階語法產生 UPLC 的既定框架稱為 <i>「Plutus-Tx」</i>，且恰好是一個 Haskell 框架。
      <br/>
      Aiken 透過提供一個能直接編譯為 UPLC 的現代化框架，改變了遊戲規則。
    </p>
  </details>

  <details>
    <summary><h3>我可以使用 Aiken 編寫鏈下/後端程式碼嗎？</h3></summary>
    <p>
      我們的主要目標是改善 Cardano 區塊鏈的
      <strong>智慧合約開發體驗</strong>。透過將 Aiken 開發為一種
      專用語言，我們可以使其功能更符合鏈上開發的
      實際需求。
      <br/>
      我們希望保持該語言的簡潔與易於管理。在去中心化
      架構中，智慧合約帶來了巨大的挑戰，單一缺陷就可能演變成
      數百萬美元的金融漏洞。此外，
      鏈上程式碼通常僅佔整個 DApp 原始碼的一小部分。
      雖然編寫合約應盡可能簡單直接，但
      智慧合約應針對審查、稽核和靜態
      分析進行優化。
      <br/>
      因此，Aiken <strong>並非</strong> 旨在成為一種通用
      語言。相反，它專注於 Cardano，並致力於提供一套高品質的
      工具組，以便自信地開發可靠的智慧合約。
    </p>
  </details>
