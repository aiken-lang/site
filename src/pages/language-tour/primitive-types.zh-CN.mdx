# 基础类型

Aiken 有 6 种内建于语言中的基础类型，可以作为字面量 (literals) 使用：布尔值 (booleans)、整数 (integers)、字符串 (strings)、字节数组 (bytearrays)、数据 (data) 和空值 (void)。该语言还包括一些用于将类型关联在一起的基本构建单元：列表 (lists)、元组 (tuples)、选项 (options)、配对 (pairs)...

别担心，我们稍后会在手册中学习如何创建您自己的[自定义类型](/language-tour/custom-types)。

<Callout type="info">
  行内注释使用 `//` 表示。我们将在整份指南的示例中使用它们来说明某些表达式的值。
</Callout>

## 布尔值 (Bool)

`Bool{:ak}` 是一个布尔值，可以是 `True{:ak}` 或 `False{:ak}`。

Aiken 定义了一些处理布尔值的运算符。毫无疑问，它们看起来会非常眼熟。

| 运算符 | 描述 | 优先级 |
| :-------: | ------------------------------------------------------------------------------------- | ---------- |
| `=={:ak}` | 相等 | 4 |
| `&&{:ak}` | 逻辑与 (AND) | 3 |
|  `\|\|`   | 逻辑或 (OR) | 2 |
| `!{:ak}`  | 逻辑非 (NOT) | 1 |
| `?{:ak}`  | 如果为假则追踪 (另请参见[疑难排解](/language-tour/troubleshooting#-operator)) | 1 |

与大多数语言一样，`&&{:ak}` 和 `||{:ak}` 是「右结合」的，这意味着如果第一个操作数已经给出了答案，它们就不会评估第二个操作数（即短路评估）。

### `and`/`or` 关键字

您很快就会发现，在构建验证器时，长的逻辑运算符链是非常常见的。

```ak
True && False && True || False
```

在上面的示例中，思考一下就能得出最终的布林值，但这也要求您仔细考虑优先级如何影响表达式分组。虽然这个特定示例可能很简单，但在实践中，这些 `True{:ak}` 和 `False{:ak}` 字面量实际上会是类型为 `Bool` 的表达式。考虑到这类布尔检查是验证器逻辑的重要基石，我们引入了关键字来增加这些所谓的关系「逻辑运算符链」的可读性。

```ak
and {
  True,
  False,
  or {
    True,
    False,
  }
}
```

使用这些关键字，分组变得一目了然，从而使逻辑运算符链更易于阅读和维护。这在组合大量逻辑运算符（例如四个或更多）时尤其出色，但在只有一两个逻辑运算符时影响较小。

## 整数 (Int)

Aiken 唯一的数字类型是任意大小的整数。这意味着没有下溢 (underflow) 或上溢 (overflow)。

```aiken
42
14
1337
```

字面量也可以使用 `_` 作为分隔符来增强可读性：

```ak
1_000_000
```

Aiken 还支持以十进制以外的进制编写整数字面量。二进制、八进制和十六进制整数分别以 `0b`、`0o` 和 `0x` 开头。

```aiken
0b00001111 == 15
0o17 == 15
0xF == 15
```

Aiken 有几个处理整数的二元算术运算符。

| 运算符 | 描述 | 优先级 |
| :------: | --------------------------- | ---------- |
| `+{:ak}` | 算术加法 | 6 |
| `-{:ak}` | 算术减法 | 6 |
| `/{:ak}` | 整除 | 7 |
| `*{:ak}` | 算术乘法 | 7 |
| `%{:ak}` | 取余数 | 7 |

当然，整数也是可以比较的，因此它们也适用于多种二元逻辑运算符：

| 运算符 | 描述 | 优先级 |
| :-------: | ---------------- | ---------- |
| `=={:ak}` | 相等 | 4 |
| `>{:ak}`  | 大于 | 4 |
| `<{:ak}`  | 小于 | 4 |
| `>={:ak}` | 大于或等于 | 4 |
| `<={:ak}` | 小于或等于 | 4 |

<Callout type="info">
在 Aiken 中，任何**可序列化**的类型都可以通过 `==` 进行比较。可序列化类型代表本指南中的任何基础类型（`Fuzzer` 和 `MillerLoopResult` 除外），以及任何由可序列化类型组成的用户定义自定义数据类型。

换句话说，您可以使用 `==` 运算符比较字节字符串 (bytestrings)、布尔值、整数、列表、元组等。
</Callout>

## 字节数组 (ByteArray)

`ByteArray` 正如其名，是一个字节数组。Aiken 支持三种符号来声明 ByteArray 字面量。

#### 作为字节数组

首先，作为范围从 0 到 255 的整数（即字节）列表：

```aiken
#[10, 255]
#[1, 256] // 会导致解析错误，因为 256 超出 1 个字节的范围
```

整数字面量的语法规则同样适用于字节数组。因此，以下语法是完全有效的：

```aiken
#[0xff, 0x42]
```

#### 作为字节字符串

其次，作为 UTF-8 编码的字节字符串。这通常是底层表示常见文本字符串的方式。在 Aiken 中，只需为此使用双引号：

```aiken
"foo" == #[0x66, 0x6f, 0x6f] == #[102, 111, 111]
```

#### 作为十六进制编码的字节字符串

因为在区块链上下文中操作 Base16 编码的字节字符串（例如交易 ID、政策 ID 等）非常常见，所以 Aiken 还支持一种简写语法，将字节数组声明为十六进制字符串。

在幕后，Aiken 会为您解码该编码字符串，并仅将原始字节存储为 ByteArray。这是通过在双引号字节字符串前加上 `#` 来实现的，如下所示：

```aiken
#"666f6f" == #[0x66, 0x6f, 0x6f] == #[102, 111, 111] == "foo"
```

注意这与以下内容有何不同：

```aiken
"666f6f" == #[0x36, 0x36, 0x36, 0x66, 0x36, 0x66] == #[54, 54, 54, 102, 54, 54]
```

## 列表 (List)

列表是值的有序集合。它们是 Aiken 中最常见的数据结构之一。

与元组不同，列表中的所有元素必须具有相同的类型。尝试使用多种不同的类型创建列表将导致类型错误。

```aiken
[1, 2, 3, 4]  // List<Int>

["text", 3, 4]  // 类型错误！
```

在列表的前端插入速度非常快，是添加新值的首选方式。

```aiken
[1, ..[2, 3]] // [1, 2, 3]
```

请注意，Aiken 中的所有数据结构都是不可变的，因此在列表前端添加元素不会更改原始列表。相反，它会高效地创建一个包含新添加元素的新列表。

```aiken
let x = [2, 3]
let y = [1, ..x]

x // [2, 3]
y // [1, 2, 3]
```

## 元组 (Tuple)

Aiken 有元组，这对于将值分组很有用。与列表不同，元组中的每个元素可以具有不同的类型。

```aiken
(10, "hello") // 类型为 (Int, ByteArray)
(1, 4, [0]) // 类型为 (Int, Int, List<Int>)
```

通常不鼓励使用长元组（即超过 3 个元素）。事实上，元组是匿名构造函数，虽然它们使用方便快捷，但往往会妨碍可读性。当类型变得更复杂时，应该使用具名记录 (records)（我们稍后会看到）。

元组的元素可以使用点（`.`）后跟元素的索引（序数）来访问。例如：

```aiken
let point = (14, 42, 1337, 0)
let a = point.1st
let b = point.2nd
let c = point.3rd
let d = point.4th
(c, d, b, a) // (1337, 0, 42, 14)
```

### 配对 (Pair)

Aiken 有一种专门用于表示两个不一定相等的类型的值配对。因此，`Pair<a, b>{:ak}` 类似于二元组 `(a, b){:ak}`。与元组一样，您可以使用序数语法访问配对中的元素：

```aiken
let foo = Pair(14, "aiken")

foo.1st == 14

foo.2nd == "aiken"
```

那么为什么要为配对专门设置一个类型呢？答案有两方面：

1. 这是在底层 Plutus 虚拟机上也存在的一种特定项。因此，鉴于 Aiken 与该机器的表示法非常接近，在需要时有表示这些项的方法是很有用甚至必要的。

2. 在合约边界，关于如何在 CBOR 中序列化配对列表存在歧义。一种方法是使用一个 CBOR 数组，其元素也是数组（每个内部数组有两个元素）；另一种方法是使用 CBOR 映射 (map)。虽然前者更直观，但事实上账本在许多内部类型中使用的是后者。在 Aiken 的第一个版本中，编译器会在序列化函数中隐式地将配对列表视为 CBOR 映射，以模拟账本。但在涉及用户定义数据类型的许多场景中，这已被证明是不便且令人困惑的，因此引入了配对。

所以作为经验法则，除非您特别想将外部类型（在 datum 和 redeemer 中）序列化为 CBOR 映射，否则您可能永远不需要使用 `Pair<a, b>{:ak}`，只需使用二元组 `(a, b){:ak}` 即可。

## 选项 (Option)

我们将 `Option{:ak}` 定义为一个具有两个构造函数的泛型类型（有关此语法的详细信息，请参见[自定义类型](/language-tour/custom-types)）：

```ak
type Option<a> {
  Some(a)
  None
}
```

类型参数 `a{:ak}` 表示 `Option{:ak}` 可以包含任何其他类型。函数可以在不假设此类型是什么的情况下操作它，或者可能要求将该类型实例化为特定的具体类型。

选项用于函数必须返回选填值的情况。例如：

```ak
/// 提取列表第一个元素（如果存在）。
fn get_head(xs: List<a>) -> Option<a> {
  when xs is {
    [] -> None
    [head, ..] -> Some(head)
  }
}

/// 确保数字是严格的正数。
fn to_positive(n: Int) -> Option<Int> {
  if n <= 0 {
    None
  } else {
    Some(n)
  }
}
```

`Option{:ak}` 类型在 Aiken 中是现成可用的；它是默认可用的类型和构造函数之一。请毫不犹豫地使用它！

### 永不 (Never)

在某些极少数情况下，需要引用一个只能是 `None` 的 `Option`。这出现在标准库的一些部分，主要是由于账本中不可预见的错误和向后兼容性考虑。对于这些场景，我们引入了 `Never` 类型，它具有一个同名的构造函数。它在所有方面都与 `None` 相同，并且会序列化为相同的二进制结构。换句话说，我们有：

```aiken
let some: Data = None
let never: Data = Never
some == never
```

## 排序 (Ordering)

[`Ordering`](https://aiken-lang.github.io/prelude/aiken.html#Ordering) 类型在比较相同类型的两个值时很有用。它的定义如下：

```aiken
pub type Ordering {
  Less
  Equal
  Greater
}
```

[标准库](https://aiken-lang.github.io/stdlib/)通常会定义对应于不同类型的 `compare` 函数，例如，要比较两个 `ByteArray`，可以使用 [`bytearray.compare`](https://aiken-lang.github.io/stdlib/aiken/primitive/bytearray.html#compare) 函数。

## 空值 (Void)

`Void{:ak}` 是一个代表零元构造函数的类型，简单来说，就是没有值。它被标记为 `Void` 作为类型和构造函数。从根本上说，如果您以元组的角度来思考，`Void{:ak}` 就是一个没有元素的元组。

`Void{:ak}` 在某些情况下很有用，但由于在 Aiken 中一切都是有类型的表达式（没有「语句」之说），您很少会遇到需要它的情况。

## 数据 (Data)

`Data{:ak}` 是一种不透明的复合类型，可以表示 Aiken 中任何可能的用户定义类型。我们将在涵盖[自定义类型](/language-tour/custom-types)时详细讨论 `Data{:ak}`。目前，请将 _Data_ 视为一种通配符，它可能代表 *任何可序列化* 的值。

当您需要在同型结构中使用来自不同类型的值时，这非常有用。任何用户定义的类型都可以转换为 `Data{:ak}`，并且您可以尝试以安全的方式将 _Data_ 转换为任何[自定义类型](/language-tour/custom-types)。此外，一些语言内建功能仅处理 `Data{:ak}`，以此作为处理多态性的一种方式。

## 字符串 (String)

在 Aiken 中，文本字符串可以写在双引号之间，并加上 `@` 前缀。

```aiken
@"Hello, Aiken!"
```

它们可以跨越多行。

```aiken
@"Hello
Aiken!"
```

在底层，文本字符串是 [UTF-8](https://en.wikipedia.org/wiki/UTF-8) 编码的二进制数据，可以包含任何有效的 Unicode。

```aiken
@"🌘 アルバイト Aiken 🌒"
```

<Callout type="warning">
  请注意，在 Aiken 和链上代码中，字符串的使用场景极其有限。它们仅用于「追踪 (tracing)」，有点像附在程序特定执行路径上的标签。例如，在验证器公开的接口中找不到它们。因此，大多数时候您可能想使用 `ByteArray` 代替，仅在除错时才使用 `String`。
</Callout>

## 高级话题

<Callout emoji="💡">
如果您刚刚**开始使用** Aiken，您可以**跳过**以下部分，因为它描述了一些更高级的类型和行为。以后有需要时随时可以回来阅读。
</Callout>

### 配对列表 (Pairs)

关联列表（亦称 Pairs）只是一个类型别名，其定义为：`type Pairs<a, b> = List<Pair<a, b>>{:ak}`。它的存在是为了方便考量，因为配对主要以这种形式出现在验证器的脚本上下文中。除对 `List{:ak}` 的所有函数（这些函数同样可以用于 `Pairs{:ak}`）之外，标准库还提供了一个专门为其量身定制的[辅助模块](https://aiken-lang.github.io/stdlib/aiken/collection/pairs.html)。

### PRNG & Fuzzer (伪随机数生成与模糊测试)

Aiken 整合了基于属性的测试 (property-based testing) 框架。是的，您没看错。基于属性的测试在这里是一等公民。如果您想了解更多相关信息，我们强烈建议参考[关于基于属性测试的指南](/language-tour/tests#property-based-test)；它将教您开始使用该框架所需的一切知识。

为了支持该框架，prelude 带有两个预定义类型。`PRNG{:ak}` 代表伪随机数生成器。它的定义是不透明的，仅对测试框架的内部有关。如果您对细节仍感兴趣，请前往 [aiken-lang/fuzz](https://github.com/aiken-lang/fuzz.git)！

类似地，`Fuzzer{:ak}` 是构建随机生成器的接口。它为多种类型提供了一组基础原语，并能很好地相互组合。

### G1Element, G2Element & MillerLoopResult

这些是 BLS12-381 加密原语特有的类型，`G1Element{:ak}`、`G2Element{:ak}` 和 `MillerLoopResult{:ak}` 捕获了某些内建函数的各种操作数和返回值的类型。它们的用法与 Aiken 本身无关，在 [CIP-0381：Plutus 对 BLS12-381 配对的支持](https://cips.cardano.org/cip/CIP-0381)中更有详细描述。
