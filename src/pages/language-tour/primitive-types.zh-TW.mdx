# 基礎型別

Aiken 有 6 種內建於語言中的基礎型別，可以作為字面量 (literals) 使用：布林值 (booleans)、整數 (integers)、字串 (strings)、字節陣列 (bytearrays)、資料 (data) 和空值 (void)。該語言還包括一些用於將型別關聯在一起的基本建構單元：列表 (lists)、元組 (tuples)、選項 (options)、配對 (pairs)...

別擔心，我們稍後會在手冊中學習如何建立您自己的[自定義型別](/language-tour/custom-types)。

<Callout type="info">
  行內註解使用 `//` 表示。我們將在整份指南的範例中使用它們來說明某些表達式的值。
</Callout>

## 布林值 (Bool)

`Bool{:ak}` 是一個布林值，可以是 `True{:ak}` 或 `False{:ak}`。

Aiken 定義了一些處理布林值的運算子。毫無疑問，它們看起來會非常眼熟。

| 運算子 | 描述 | 優先級 |
| :-------: | ------------------------------------------------------------------------------------- | ---------- |
| `=={:ak}` | 相等 | 4 |
| `&&{:ak}` | 邏輯與 (AND) | 3 |
|  `\|\|`   | 邏輯或 (OR) | 2 |
| `!{:ak}`  | 邏輯非 (NOT) | 1 |
| `?{:ak}`  | 如果為假則追蹤 (另請參見[疑難排解](/language-tour/troubleshooting#-operator)) | 1 |

與大多數語言一樣，`&&{:ak}` 和 `||{:ak}` 是「右結合」的，這意味著如果第一個運算數已經給出了答案，它們就不會評估第二個運算數（即短路評估）。

### `and`/`or` 關鍵字

您很快就會發現，在構建驗證器時，長的邏輯運算子鏈是非常常見的。

```ak
True && False && True || False
```

在上面的範例中，思考一下就能得出最終的布林值，但這也要求您仔細考慮優先級如何影響表達式分組。雖然這個特定範例可能很簡單，但在實踐中，這些 `True{:ak}` 和 `False{:ak}` 字面量實際上會是型別為 `Bool` 的表達式。考慮到這類布林檢查是驗證器邏輯的重要基石，我們引入了關鍵字來增加這些所謂的「邏輯運算子鏈」的可讀性。

```ak
and {
  True,
  False,
  or {
    True,
    False,
  }
}
```

使用這些關鍵字，分組變得一目瞭然，從而使邏輯運算子鏈更易於閱讀和維護。這在組合大量邏輯運算子（例如四個或更多）時尤其出色，但在只有一兩個邏輯運算子時影響較小。

## 整數 (Int)

Aiken 唯一的數字型別是任意大小的整數。這意味著沒有下溢 (underflow) 或上溢 (overflow)。

```aiken
42
14
1337
```

字面量也可以使用 `_` 作為分隔符號來增強可讀性：

```ak
1_000_000
```

Aiken 還支援以十進位以外的進位編寫整数字面量。二進位、八進位和十六進位整數分別以 `0b`、`0o` 和 `0x` 開頭。

```aiken
0b00001111 == 15
0o17 == 15
0xF == 15
```

Aiken 有幾個處理整數的二元算術運算子。

| 運算子 | 描述 | 優先級 |
| :------: | --------------------------- | ---------- |
| `+{:ak}` | 算術加法 | 6 |
| `-{:ak}` | 算術減法 | 6 |
| `/{:ak}` | 整除 | 7 |
| `*{:ak}` | 算術乘法 | 7 |
| `%{:ak}` | 取餘數 | 7 |

當然，整數也是可以比較的，因此它們也適用於多種二元邏輯運算子：

| 運算子 | 描述 | 優先級 |
| :-------: | ---------------- | ---------- |
| `=={:ak}` | 相等 | 4 |
| `>{:ak}`  | 大於 | 4 |
| `<{:ak}`  | 小於 | 4 |
| `>={:ak}` | 大於或等於 | 4 |
| `<={:ak}` | 小於或等於 | 4 |

<Callout type="info">
在 Aiken 中，任何**可序列化**的型別都可以透過 `==` 進行比較。可序列化型別代表本指南中的任何基礎型別（`Fuzzer` 和 `MillerLoopResult` 除外），以及任何由可序列化型別組成的使用者定義自定義資料型別。

換句話說，您可以使用 `==` 運算子比較位元組字串 (bytestrings)、布林值、整數、列表、元組等。
</Callout>

## 字節陣列 (ByteArray)

`ByteArray` 正如其名，是一個位元組陣列。Aiken 支援三種符號來聲明 ByteArray 字面量。

#### 作為位元組陣列

首先，作為範圍從 0 到 255 的整數（即位元組）列表：

```aiken
#[10, 255]
#[1, 256] // 會導致解析錯誤，因為 256 超出 1 個位元組的範圍
```

整數字面量的語法規則同樣適用於字節陣列。因此，以下語法是完全有效的：

```aiken
#[0xff, 0x42]
```

#### 作為位元組字串

其次，作為 UTF-8 編碼的位元組字串。這通常是底層表示常見文本字串的方式。在 Aiken 中，只需為此使用雙引號：

```aiken
"foo" == #[0x66, 0x6f, 0x6f] == #[102, 111, 111]
```

#### 作為十六進位編碼的位元組字串

因為在區塊鏈上下文中操作 Base16 編碼的位元組字串（例如交易 ID、政策 ID 等）非常常見，所以 Aiken 還支援一種簡寫語法，將位元組陣列聲明為十六進位字串。

在幕後，Aiken 會為您解碼該編碼字串，並僅將原始位元組儲存為 ByteArray。這是透過在雙引號位元組字串前加上 `#` 來實現的，如下所示：

```aiken
#"666f6f" == #[0x66, 0x6f, 0x6f] == #[102, 111, 111] == "foo"
```

注意這與以下內容有何不同：

```aiken
"666f6f" == #[0x36, 0x36, 0x36, 0x66, 0x36, 0x66] == #[54, 54, 54, 102, 54, 54]
```

## 列表 (List)

列表是值的有序集合。它們是 Aiken 中最常見的資料結構之一。

與元組不同，列表中的所有元素必須具有相同的型別。嘗試使用多種不同的型別建立列表將導致型別錯誤。

```aiken
[1, 2, 3, 4]  // List<Int>

["text", 3, 4]  // 型別錯誤！
```

在列表的前端插入速度非常快，是添加新值的首選方式。

```aiken
[1, ..[2, 3]] // [1, 2, 3]
```

請注意，Aiken 中的所有資料結構都是不可變的，因此在列表前端添加元素不會更改原始列表。相反，它會高效地建立一個包含新添加元素的新列表。

```aiken
let x = [2, 3]
let y = [1, ..x]

x // [2, 3]
y // [1, 2, 3]
```

## 元組 (Tuple)

Aiken 有元組，這對於將值分組很有用。與列表不同，元組中的每個元素可以具有不同的型別。

```aiken
(10, "hello") // 型別為 (Int, ByteArray)
(1, 4, [0]) // 型別為 (Int, Int, List<Int>)
```

通常不鼓勵使用長元組（即超過 3 個元素）。事實上，元組是匿名構造函數，雖然它們使用方便快捷，但往往會妨礙可讀性。當型別變得更複雜時，應該使用具名記錄 (records)（我們稍後會看到）。

元組的元素可以使用點（`.`）後跟元素的索引（序數）來存取。例如：

```aiken
let point = (14, 42, 1337, 0)
let a = point.1st
let b = point.2nd
let c = point.3rd
let d = point.4th
(c, d, b, a) // (1337, 0, 42, 14)
```

### 配對 (Pair)

Aiken 有一種專門用於表示兩個不一定相等的型別的值配對。因此，`Pair<a, b>{:ak}` 類似於二元組 `(a, b){:ak}`。與元組一樣，您可以使用序數語法存取配對中的元素：

```aiken
let foo = Pair(14, "aiken")

foo.1st == 14

foo.2nd == "aiken"
```

那麼為什麼要為配對專門設置一個型別呢？答案有兩方面：

1. 這是在底層 Plutus 虛擬機上也存在的一種特定項。因此，鑒於 Aiken 與該機器的表示法非常接近，在需要時有表示這些項的方法是很有用甚至必要的。

2. 在合約邊界，關於如何在 CBOR 中序列化配對列表存在歧義。一種方法是使用一個 CBOR 陣列，其元素也是陣列（每個內部陣列有兩個元素）；另一種方法是使用 CBOR 映射 (map)。雖然前者更直觀，但事實上帳本在許多內部型別中使用的是後者。在 Aiken 的第一個版本中，編譯器會在序列化函數中隱式地將配對列表視為 CBOR 映射，以模擬帳本。但在涉及使用者定義資料型別的許多場景中，這已被證明是不便且令人困惑的，因此引入了配對。

所以作為經驗法則，除非您特別想將外部型別（在 datum 和 redeemer 中）序列化為 CBOR 映射，否則您可能永遠不需要使用 `Pair<a, b>{:ak}`，只需使用二元組 `(a, b){:ak}` 即可。

## 選項 (Option)

我們將 `Option{:ak}` 定義為一個具有兩個構造函數的泛型型別（有關此語法的詳細資訊，請參見[自定義型別](/language-tour/custom-types)）：

```ak
type Option<a> {
  Some(a)
  None
}
```

型別參數 `a{:ak}` 表示 `Option{:ak}` 可以包含任何其他型別。函數可以在不假設此型別是什麼的情況下操作它，或者可能要求將該型別實例化為特定的具體型別。

選項用於函數必須返回選填值的情況。例如：

```ak
/// 提取列表的第一個元素（如果存在）。
fn get_head(xs: List<a>) -> Option<a> {
  when xs is {
    [] -> None
    [head, ..] -> Some(head)
  }
}

/// 確保數字是嚴格的正數。
fn to_positive(n: Int) -> Option<Int> {
  if n <= 0 {
    None
  } else {
    Some(n)
  }
}
```

`Option{:ak}` 型別在 Aiken 中是現成可用的；它是預設可用的型別和構造函數之一。請毫不猶豫地使用它！

### 永不 (Never)

在某些極少數情況下，需要引用一個只能是 `None` 的 `Option`。這出現在標準庫的一些部分，主要是由於帳本中不可預見的錯誤和向後相容性考慮。對於這些場景，我們引入了 `Never` 型別，它具有一個同名的構造函數。它在所有方面都與 `None` 相同，並且會序列化為相同的二進位結構。換句話說，我們有：

```aiken
let some: Data = None
let never: Data = Never
some == never
```

## 排序 (Ordering)

[`Ordering`](https://aiken-lang.github.io/prelude/aiken.html#Ordering) 型別在比較相同型別的兩個值時很有用。它的定義如下：

```aiken
pub type Ordering {
  Less
  Equal
  Greater
}
```

[標準庫](https://aiken-lang.github.io/stdlib/)通常會定義對應於不同型別的 `compare` 函數，例如，要比較兩個 `ByteArray`，可以使用 [`bytearray.compare`](https://aiken-lang.github.io/stdlib/aiken/primitive/bytearray.html#compare) 函數。

## 空值 (Void)

`Void{:ak}` 是一個代表零元構造函數的型別，簡單來說，就是沒有值。它被標記為 `Void` 作為型別和構造函數。從根本上說，如果您以元組的角度來思考，`Void{:ak}` 就是一個沒有元素的元組。

`Void{:ak}` 在某些情況下很有用，但由於在 Aiken 中一切都是有型別的表達式（沒有「語句」之說），您很少會遇到需要它的情況。

## 資料 (Data)

`Data{:ak}` 是一種不透明的複合型別，可以表示 Aiken 中任何可能的使用者定義型別。我們將在涵蓋[自定義型別](/language-tour/custom-types)時詳細討論 `Data{:ak}`。目前，請將 _Data_ 視為一種通配符，它可能代表 *任何可序列化* 的值。

當您需要在同型結構中使用來自不同型別的值時，這非常有用。任何使用者定義的型別都可以轉換為 `Data{:ak}`，並且您可以嘗試以安全的方式將 _Data_ 轉換為任何[自定義型別](/language-tour/custom-types)。此外，一些語言內建功能僅處理 `Data{:ak}`，以此作為處理多型性的一種方式。

## 字串 (String)

在 Aiken 中，文本字串可以寫在雙引號之間，並加上 `@` 前綴。

```aiken
@"Hello, Aiken!"
```

它們可以跨越多行。

```aiken
@"Hello
Aiken!"
```

在底層，文本字串是 [UTF-8](https://en.wikipedia.org/wiki/UTF-8) 編碼的二進位資料，可以包含任何有效的 Unicode。

```aiken
@"🌘 アルバイト Aiken 🌒"
```

<Callout type="warning">
  請注意，在 Aiken 和鏈上程式碼中，字串的使用場景極其有限。它們僅用於「追蹤 (tracing)」，有點像附在程式特定執行路徑上的標籤。例如，在驗證器公開的介面中找不到它們。因此，大多數時候您可能想使用 `ByteArray` 代替，僅在除錯時才使用 `String`。
</Callout>

## 高級話題

<Callout emoji="💡">
如果您剛剛**開始使用** Aiken，您可以**跳過**以下部分，因為它描述了一些更高級的型別和行為。以後有需要時隨時可以回來閱讀。
</Callout>

### 配對列表 (Pairs)

關聯列表（亦稱 Pairs）只是一個型別別名，其定義為：`type Pairs<a, b> = List<Pair<a, b>>{:ak}`。它的存在是為了方便考量，因為配對主要以這種形式出現在驗證器的腳本上下文中。除對 `List{:ak}` 的所有函數（這些函數同樣可以用於 `Pairs{:ak}`）之外，標準庫還提供了一個專門為其量身定制的[輔助模組](https://aiken-lang.github.io/stdlib/aiken/collection/pairs.html)。

### PRNG & Fuzzer (偽隨機數生成與模糊測試)

Aiken 整合了基於屬性的測試 (property-based testing) 框架。是的，您沒看錯。基於屬性的測試在這裡是一等公民。如果您想瞭解更多相關資訊，我們強烈建議參考[關於基於屬性測試的指南](/language-tour/tests#property-based-test)；它將教您開始使用該框架所需的一切知識。

為了支持該框架，prelude 帶有兩個預定義型別。`PRNG{:ak}` 代表偽隨機數生成器。它的定義是不透明的，僅對測試框架的內部有關。如果您對細細節仍感興趣，請前往 [aiken-lang/fuzz](https://github.com/aiken-lang/fuzz.git)！

類似地，`Fuzzer{:ak}` 是構建隨機生成器的介面。它為多種型別提供了一組基礎原語，並能很好地相互組合。

### G1Element, G2Element & MillerLoopResult

這些是 BLS12-381 加密原語特有的型別，`G1Element{:ak}`、`G2Element{:ak}` 和 `MillerLoopResult{:ak}` 捕獲了某些內建函數的各種運算數和返回值的型別。它們的用法與 Aiken 本身無關，在 [CIP-0381：Plutus 對 BLS12-381 配對的支持](https://cips.cardano.org/cip/CIP-0381)中更有詳細描述。
